03 · How It Works

Complaint
to cure
in 4 steps.

ROOTS sits on top of the MCD system, watching 5,000 daily complaints for patterns no human dispatcher could spot.

01Collect from anywhere
02Cluster by space + time
03Find the root cause
04Rank by real cost
01
File from anywhere.
A domestic worker files by USSD on a feature phone. A student sends a WhatsApp photo. A building society submits via web. Every channel flows into one pipeline — in any of India's 22 scheduled languages.
WhatsApp · Web PWA · USSD *123# · Voice IVR · UMANG
Bhashini API · 22 languages · No smartphone required
02
ST-DBSCAN clusters by space + time.
Six complaints within 400 metres and 72 hours are almost certainly the same event. Spatial radius adapts to population density — tighter in dense wards, wider in sparse ones.
ε = 400m (density class 3) · T = 72h · MinPts = 3
Dynamic ε: 200m → 800m by Census density · O(n log n)
03
Causal graph finds the root failure.
The cluster is scored against 8 infrastructure failure signatures. Above 84% confidence — dispatch. Below — flag for human review. No false routing, no wasted teams.
8 signatures · Bayesian scoring · Threshold 84%
Below threshold → unclassified → manual review
04
ROI optimizer ranks by real cost.
Not most complaints — most environmental harm prevented per rupee. Greedy knapsack approximation selects the optimal repair set within 15% of the mathematical optimum.
Maximise: Σ(env_harm_prevented × xᵢ)
Subject to: Σ(cost × xᵢ) ≤ Budget · within 15% optimal
